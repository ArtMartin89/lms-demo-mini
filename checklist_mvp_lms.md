# Чеклист для команды разработчиков MVP LMS с тестированием

**Формат: Markdown. Структура: критерий → acceptance criteria → метрики
для контроля.**

## 1. Проверка гипотезы и востребованности (Product-Market Fit)

### Acceptance criteria

-   [ ] В продукт встроены минимальные метрики по флагманской гипотезе:
    **создатель теста может создать тест ≤ 30 минут**, тестируемый ---
    пройти его без ошибок.\
-   [ ] В интерфейсе доступна минимальная онбординг-подсказка (1--2
    шага).\
-   [ ] Предусмотрен сбор пользовательской обратной связи (формы,
    быстрые оценки).\
-   [ ] На этапе регистрации/первых шагов собирается информация о роли:
    создатель контента / студент / HR.

### Метрики

-   **Активация (AHA):** ≥ 40% новых пользователей создали хотя бы один
    тест или прошли тест.\
-   **Retention D7:** ≥ 15%.\
-   **Completion rate тестов:** ≥ 70%.\
-   **Время создания теста:** медиана ≤ 30 минут.

## 2. Стабильность критического пользовательского пути

### Acceptance criteria

-   [ ] Создание теста не вызывает необработанных ошибок.\
-   [ ] Сохранение вопросов/ответов работает транзакционно.\
-   [ ] Прохождение теста не «роняет» ответы при нестабильной сети.\
-   [ ] Понятные сообщения об ошибках.\
-   [ ] Нагрузочный тест на 20--50 параллельных сессий.

### Метрики

-   **CRITICAL FLOW ERROR RATE:** ≤ 1%.\
-   **Доля потерянных ответов:** 0%.\
-   **Время сохранения шага:** ≤ 500 мс.\
-   **Uptime backend:** ≥ 99%.

## 3. UX создающего и проходящего тест

### Acceptance criteria

-   [ ] Флоу создания теста ≤ 3 экранов.\
-   [ ] Предпросмотр теста.\
-   [ ] Не более 3 типов вопросов.\
-   [ ] Прогресс и таймер при прохождении.\
-   [ ] Юзабилити-сессии и исправление критичных проблем.

### Метрики

-   **ABANDON RATE создания:** ≤ 30%.\
-   **ABANDON RATE прохождения:** ≤ 15%.\
-   **Время до первого вопроса:** ≤ 15 сек.\
-   **NPS:** ≥ 0.

## 4. Честность и валидность результатов

### Acceptance criteria

-   [ ] Уникальные ссылки.\
-   [ ] Логирование действий.\
-   [ ] Таймер.\
-   [ ] Предупреждения о смене вкладок.\
-   [ ] В отчёте --- время прохождения, попытки, подозрительная
    активность.

### Метрики

-   **Аномально короткие прохождения:** ≤ 5%.\
-   **Многовкладочные сессии:** ≤ 20%.\
-   **Ошибки подсчёта баллов:** 0%.

## 5. Архитектура и масштабируемость

### Acceptance criteria

-   [ ] Модульное разделение.\
-   [ ] Нормализованная БД.\
-   [ ] Миграции.\
-   [ ] Мониторинг и логирование.\
-   [ ] Возможность горизонтального масштабирования.\
-   [ ] Простые фич-флаги.

### Метрики

-   **API P95:** ≤ 300--500 мс.\
-   **DB-load при нагрузке:** CPU ≤ 60%, задержки ≤ 200 мс.\
-   **Логируемость:** ≥ 95%.\
-   **Время деплоя:** ≤ 5 минут.
